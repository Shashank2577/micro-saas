# Detailed Specification for BudgetPilot

## Overview
**App:** BudgetPilot
**Cluster:** Cluster C
**Problem:** Budgets are built annually in Excel, reviewed quarterly, and are obsolete by month 2. Budget vs. actual reports are manual and tell you what happened, not what to do.
**Solution:** AI budgeting co-pilot. Generates budget proposals from historical data and business goals. Tracks variance in real time with natural language explanations ("Engineering is 12% over due to unplanned AWS costs from the new ML pipeline"). Alerts on projected overruns before leadership reviews.
**AI Pattern:** Co-pilot + pipeline (continuous variance monitoring).
**Monetization:** $199/month

## Database Schema (PostgreSQL with pgvector & Tenant Isolation)

```sql
CREATE TABLE budgets (
    id UUID PRIMARY KEY,
    tenant_id UUID NOT NULL,
    name VARCHAR(255) NOT NULL,
    fiscal_year INT NOT NULL,
    total_amount DECIMAL(15, 2) NOT NULL,
    status VARCHAR(50) NOT NULL, -- DRAFT, ACTIVE, ARCHIVED
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE budget_items (
    id UUID PRIMARY KEY,
    tenant_id UUID NOT NULL,
    budget_id UUID NOT NULL REFERENCES budgets(id) ON DELETE CASCADE,
    category VARCHAR(100) NOT NULL,
    department VARCHAR(100) NOT NULL,
    allocated_amount DECIMAL(15, 2) NOT NULL,
    actual_amount DECIMAL(15, 2) DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE expenses (
    id UUID PRIMARY KEY,
    tenant_id UUID NOT NULL,
    budget_item_id UUID NOT NULL REFERENCES budget_items(id) ON DELETE CASCADE,
    amount DECIMAL(15, 2) NOT NULL,
    description TEXT,
    date DATE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE variances (
    id UUID PRIMARY KEY,
    tenant_id UUID NOT NULL,
    budget_item_id UUID NOT NULL REFERENCES budget_items(id) ON DELETE CASCADE,
    variance_amount DECIMAL(15, 2) NOT NULL,
    variance_percentage DECIMAL(5, 2) NOT NULL,
    explanation TEXT, -- Generated by AI
    date_calculated TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Indexes for performance and multi-tenancy
CREATE INDEX idx_budgets_tenant ON budgets(tenant_id);
CREATE INDEX idx_budget_items_tenant_budget ON budget_items(tenant_id, budget_id);
CREATE INDEX idx_expenses_tenant_item ON expenses(tenant_id, budget_item_id);
CREATE INDEX idx_variances_tenant_item ON variances(tenant_id, budget_item_id);
```

## REST API Endpoints

### Budgets
- `GET /api/v1/budgets`: List budgets for tenant
- `POST /api/v1/budgets`: Create a new budget
- `GET /api/v1/budgets/{id}`: Get budget details
- `PUT /api/v1/budgets/{id}`: Update budget
- `DELETE /api/v1/budgets/{id}`: Delete budget

### Budget Items
- `GET /api/v1/budgets/{budgetId}/items`: List items for a budget
- `POST /api/v1/budgets/{budgetId}/items`: Add item to budget
- `PUT /api/v1/items/{id}`: Update budget item
- `DELETE /api/v1/items/{id}`: Delete budget item

### AI Integrations
- `POST /api/v1/ai/propose-budget`: Generate a budget proposal based on goals and historical data.
- `POST /api/v1/ai/explain-variance/{varianceId}`: Generate a natural language explanation for a specific variance.

## AI Integrations (LiteLLM)

1. **Variance Explanation:**
   - **Trigger:** Variance exceeds a predefined threshold (e.g., 10%).
   - **Prompt:** "Explain the variance for budget item {item}. Allocated: {allocated}, Actual: {actual}. Recent expenses: {expenses}. Provide a concise explanation for the overrun."
2. **Budget Proposal:**
   - **Trigger:** User requests a new budget proposal in the Planning UI.
   - **Prompt:** "Create a budget proposal for {department} for FY {year}. Business goals: {goals}. Historical spend: {historicalData}. Provide a breakdown of recommended allocations with reasoning."

## Acceptance Criteria & Test Cases

1. **Test Budget CRUD:** Verify budgets can be created, read, updated, and deleted, respecting tenant boundaries.
2. **Test Variance Calculation:** Verify the system correctly calculates variance percentage when expenses are added.
3. **Test AI Variance Explanation:** Mock LiteLLM response and verify `generateVarianceExplanation` returns the expected text and saves it to the variance record.
4. **Test AI Budget Proposal:** Mock LiteLLM and verify a structured budget proposal is returned based on input goals.
5. **Test Webhook Event Emission:** Verify `budget.variance.alert` is published to pgmq when variance exceeds threshold.

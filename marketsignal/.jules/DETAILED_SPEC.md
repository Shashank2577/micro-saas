# Detailed Specification: MarketSignal

## Overview
MarketSignal is an AI market intelligence platform that monitors news, earnings calls, patent filings, startup activity, job postings, and regulatory developments. It detects patterns, assigns signal strength ratings, and generates weekly market briefs.

## Architecture

### Backend Stack
- Spring Boot 3.3.5 (Java 21)
- cc-starter for cross-cutting concerns (multi-tenancy)
- PostgreSQL 16 with pgvector
- pgmq for asynchronous job processing
- LiteLLM integration for analysis, pattern detection, and brief generation.

### Frontend Stack
- Next.js 14 App Router
- React, Tailwind CSS
- TypeScript

## Entities

1. `MarketSegment`
   - `id` (UUID), `tenantId` (UUID)
   - `name` (String)
   - `description` (String)
   - `keywords` (List<String>)
   - `createdAt`, `updatedAt`

2. `InformationSource`
   - `id` (UUID), `tenantId` (UUID)
   - `name` (String)
   - `sourceType` (Enum: NEWS, EARNINGS_CALL, PATENT, STARTUP, JOB_POSTING, REGULATORY, INTERNAL)
   - `url` (String)
   - `active` (Boolean)

3. `MarketSignal`
   - `id` (UUID), `tenantId` (UUID)
   - `title` (String)
   - `content` (String)
   - `sourceId` (UUID, reference to InformationSource)
   - `publishedAt` (Timestamp)
   - `signalStrength` (Integer, 1-10)
   - `sentiment` (String)
   - `strategicImplications` (String)
   - `embedding` (Vector, pgvector)
   - `createdAt`

4. `MarketPattern`
   - `id` (UUID), `tenantId` (UUID)
   - `title` (String)
   - `description` (String)
   - `patternType` (Enum: COMPETITIVE_SHIFT, OPPORTUNITY_GAP, RISK_FLAG)
   - `signalIds` (List<UUID>)
   - `detectedAt` (Timestamp)

5. `MarketBrief`
   - `id` (UUID), `tenantId` (UUID)
   - `title` (String)
   - `summary` (String)
   - `content` (String) // Markdown generated by AI
   - `generatedAt` (Timestamp)

## APIs

- `GET /api/v1/segments` - List segments
- `POST /api/v1/segments` - Create segment
- `GET /api/v1/sources` - List sources
- `POST /api/v1/sources` - Create source
- `GET /api/v1/signals` - List signals (filterable by segment, strength, type)
- `POST /api/v1/signals` - Ingest signal
- `GET /api/v1/patterns` - List detected patterns
- `POST /api/v1/patterns/detect` - Trigger pattern detection
- `GET /api/v1/briefs` - List briefs
- `POST /api/v1/briefs/generate` - Generate brief

## Services
- `IngestionService`: Fetches or receives data from sources, extracts content, and stores signals.
- `AnalysisService`: Uses LiteLLM to analyze signals, extract sentiment, strength, and strategic implications.
- `PatternDetectionService`: Uses embeddings (pgvector) and LiteLLM to cluster signals and identify patterns (shifts, opportunities, risks).
- `BriefGenerationService`: Uses LiteLLM to summarize the top signals and patterns into a weekly market brief.

## Frontend
- **Dashboard**: High-level overview of recent signals, top patterns, and latest brief.
- **Segments & Sources**: Manage tracked segments and sources.
- **Signals Feed**: List of all signals with filtering and details.
- **Patterns View**: Visual timeline and list of competitive shifts, opportunities, and risks.
- **Briefs**: View and read generated market briefs.

## Error Handling
- Use `@ControllerAdvice` for consistent API error responses.
- AI failures should be retried and gracefully handled (circuit breaker).
- Invalid inputs will return 400 Bad Request with validation errors.
